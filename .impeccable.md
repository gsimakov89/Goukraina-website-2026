# Go Ukraina — Design Context

## Users
Non-technical charity administrators publishing field reports, managing media, and tuning SEO for a humanitarian nonprofit (Go Ukraina — Ukraine clean water, power, advocacy). Primary persona: a communications volunteer with zero CMS experience, working from a laptop between meetings, occasionally on a phone between site visits in Ukraine. Secondary persona: the founder doing quick publish-and-review on mobile.

## Brand Personality
**Three words:** resolute, humane, unflashy.

Go Ukraina operates in wartime humanitarian work — the admin is a *working tool*, not a showcase. The voice is the voice of a trusted field operations manual: plain, confident, respectful of the people on both ends (authors and readers). No marketing gloss. No "AI-generated polish" — glowing gradients, neon cyan, glassmorphism, and big emoji would cheapen the mission.

## Aesthetic Direction
**Editorial operations console.** Think *The New York Times CMS* crossed with a *Scandinavian utility panel* — dense but breathable, reverent of the content, the interface recedes so the writing stands forward.

**Theme: Light mode.** Admins work during the day, often on variable connections and public-space screens. Light surfaces read as trustworthy and professional for nonprofit work and map to print/editorial heritage. A limited dark accent (the masthead, buttons) provides weight without theatricality.

**Color:** Ukraine flag carries literal meaning here — blue (#1565C0 territory, but desaturated toward #1D3B6C for UI) and a muted gold (#C4953F, not the neon #FFD700). Neutrals tinted toward the blue hue — warm-blue near-whites, cool near-blacks. Red used only for destructive/critical states.

**Anti-references:** Any admin that looks like a SaaS landing page. Neon dashboards. Framer/Linear-clone aesthetics. Dark mode with glowing blue accents. Anything with a "gradient brand mark" or a purple-to-pink button.

## Design Principles

1. **Respect the writer.** The editor surface is the document; chrome recedes. No distracting animation while the author is typing. Margins breathe.
2. **Teach through the interface.** Every action has a plain-English sentence explaining consequences. "Rebuild" shows what it does, not just a verb.
3. **Predictable hierarchy.** One primary action per screen, one sidebar nav, one topbar. No competing CTAs.
4. **Mobile is real usage.** The founder does mobile triage between meetings. Full-capability phone layout — no "use a bigger screen" dead ends.
5. **No AI tells.** No border-left accent stripes, no gradient text, no glassmorphism, no DM Sans / Fraunces / Inter. Pair a distinctive sans (Manrope) with a quiet serif voice (Fraunces is banned — use an editorial sans only, with weight contrast for hierarchy). Monospace reserved for slugs, keys, code blocks.

## Typography
- **UI + body:** Manrope (300/500/700) — neutral but warm grotesk, reads as operational not trendy.
- **Editorial / headlines in content preview:** Spectral (variable, 400/600) — quiet transitional serif; evokes field reports and editorial voice without cliché.
- **Mono:** JetBrains Mono — slugs, URLs, keys, code previews.
- Scale: 12 / 14 / 16 / 20 / 26 / 34 px (ratio ≈ 1.3). Line height: 1.5 body, 1.25 headings.

## Breakpoints
- Base (mobile, 0-640px): single column, hamburger nav, editor full-bleed.
- `md` 768px: sidebar visible, two-pane editor (list + canvas).
- `lg` 1024px: three-pane blog editor (list + canvas + SEO rail).
- `xl` 1280px: generous margins around canvas.
